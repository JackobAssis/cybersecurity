🛡️ Miniguia de Estudos com NotebookLM: Fundamentos de Cibersegurança
📌 Contexto e Objetivos

Este projeto tem como objetivo utilizar a ferramenta NotebookLM como um “segundo cérebro” para organizar, estruturar e potencializar os estudos em cibersegurança durante o bootcamp da DIO.

A proposta é transformar o aprendizado passivo em ativo, utilizando inteligência artificial para analisar conteúdos, gerar insights e apoiar a tomada de decisões no estudo.

🎯 Objetivos principais:
Consolidar conhecimentos fundamentais em cibersegurança
Desenvolver pensamento crítico sobre vulnerabilidades e riscos
Aplicar conceitos em cenários práticos
Criar um sistema reutilizável de aprendizado com IA
Construir base para atuação profissional na área

🧠 Engenharia de Prompts e "Cicatrizes"

Durante o uso do NotebookLM, foram realizados testes com diferentes tipos de prompts para obter respostas mais úteis e aplicáveis.

🔹 Prompt 1 — Fundamentos

Prompt inicial:

“Explique cibersegurança”

Problema:
Resposta muito genérica

Prompt ajustado:

“Explique os fundamentos da cibersegurança com foco em aplicações práticas para iniciantes”

Resultado:
Resposta mais estruturada e aplicável

🔹 Prompt 2 — Vulnerabilidades

Prompt inicial:

“Quais são as principais vulnerabilidades?”

Problema:
Superficial

Prompt ajustado:

“Explique as principais vulnerabilidades web segundo a OWASP e como evitá-las”

Resultado:
Mais técnico e relevante

🔹 Prompt 3 — Aplicação prática

Prompt inicial:

“Como proteger sistemas?”

Problema:
Muito amplo

Prompt ajustado:

“Quais práticas básicas de segurança podem ser aplicadas em um sistema backend simples?”

Resultado:
Direcionado e útil

⚠️ Cicatrizes (Aprendizados)
Prompts genéricos geram respostas fracas
Contexto melhora drasticamente a qualidade
A IA funciona melhor como apoio, não substituto do estudo
Iterar prompts é essencial para obter bons resultados
📘 Miniguia de Estudo
🧠 Resumo Estruturado

A cibersegurança é o conjunto de práticas, tecnologias e processos utilizados para proteger sistemas, redes e dados contra acessos não autorizados e ataques.

🔐 Princípios básicos:
Confidencialidade → proteger dados contra acesso indevido
Integridade → garantir que os dados não sejam alterados
Disponibilidade → manter sistemas acessíveis
🧩 Principais áreas:
Segurança de redes
Segurança de aplicações
Testes de invasão (Pentest)
Engenharia social
⚠️ Principais ameaças:
Phishing
Malware
Ataques de força bruta
Exploração de vulnerabilidades
📖 Glossário
Firewall → controla o tráfego de rede
Phishing → tentativa de roubo de dados via engenharia social
Malware → software malicioso
Exploit → código que explora vulnerabilidades
Pentest → teste de invasão controlado
⚙️ Prompts Reutilizáveis
📌 Aprendizado

“Explique esse conceito de forma simples e com exemplo prático”

📌 Aplicação

“Como aplicar esse conceito em um sistema real?”

📌 Análise

“Quais riscos esse cenário apresenta?”

📌 Evolução

“Qual o próximo passo para evoluir nesse tema?”
