# 🛡️ CyberSec Second Brain: Sistema de Aprendizado em Cibersegurança com IA

---

## 📌 Contexto e Objetivos

Este projeto foi desenvolvido como parte do bootcamp de cibersegurança promovido pela Digital Innovation One (DIO) em parceria com a Riachuelo.

A proposta consiste na criação de um sistema de aprendizado contínuo baseado no conceito de “Second Brain”, utilizando a ferramenta NotebookLM para organizar, analisar e estruturar conhecimentos em cibersegurança.

O objetivo é transformar o aprendizado passivo em ativo, utilizando inteligência artificial como apoio na construção de conhecimento técnico e desenvolvimento de pensamento crítico.

### 🎯 Objetivos principais:
- Consolidar fundamentos de cibersegurança  
- Organizar conteúdos técnicos de forma estruturada  
- Desenvolver pensamento analítico voltado à segurança  
- Aplicar conceitos em cenários práticos  
- Criar um sistema reutilizável de aprendizado com IA  
- Construir base para atuação profissional na área

## 🧠 Engenharia de Prompts e "Cicatrizes"

Durante o uso do NotebookLM, foram realizados testes e ajustes de prompts para obter respostas mais relevantes.

---

### 🔹 Prompt 1 — Fundamentos

**Prompt inicial:**
> “Explique cibersegurança”

**Problema:**  
Resposta muito genérica

**Prompt ajustado:**
> “Explique os fundamentos da cibersegurança com foco em aplicações práticas para iniciantes”

**Resultado:**  
Resposta mais clara e aplicável

---

### 🔹 Prompt 2 — Vulnerabilidades

**Prompt inicial:**
> “Quais são as principais vulnerabilidades?”

**Problema:**  
Resposta superficial

**Prompt ajustado:**
> “Explique as principais vulnerabilidades web com base no OWASP Top 10 e como evitá-las”

**Resultado:**  
Mais técnico e útil

---

### 🔹 Prompt 3 — Aplicação prática

**Prompt inicial:**
> “Como proteger sistemas?”

**Problema:**  
Muito amplo

**Prompt ajustado:**
> “Quais práticas básicas de segurança podem ser aplicadas em um sistema backend simples?”

**Resultado:**  
Direcionado e aplicável

---

### 🔹 Prompt Avançado

> “Analise este conteúdo com foco em identificação de vulnerabilidades e proponha medidas de mitigação”

👉 Resultado: respostas mais próximas de cenários reais

---

### ⚠️ Cicatrizes (Aprendizados)

- Prompts genéricos geram respostas fracas  
- Contexto melhora significativamente os resultados  
- Iteração de prompts é essencial  
- A IA funciona melhor como apoio, não substituto do estudo

## 🔍 Simulação de Aplicação Prática

Com base nos estudos realizados, foram analisados cenários de vulnerabilidades comuns descritos pelo OWASP Top 10, incluindo:

- Injeção (SQL Injection)  
- Falhas de autenticação  
- Exposição de dados sensíveis  

Para cada cenário, foram considerados:
- possíveis riscos  
- impactos no sistema  
- medidas básicas de mitigação

## 📘 Miniguia de Estudo

---

### 🧠 Resumo Estruturado

A cibersegurança é o conjunto de práticas, tecnologias e processos utilizados para proteger sistemas, redes e dados.

#### 🔐 Princípios básicos:
- **Confidencialidade** → proteção contra acesso não autorizado  
- **Integridade** → garantia de que os dados não foram alterados  
- **Disponibilidade** → acesso contínuo aos sistemas  

#### 🧩 Principais áreas:
- Segurança de redes  
- Segurança de aplicações  
- Testes de invasão (Pentest)  
- Engenharia social  

#### ⚠️ Principais ameaças:
- Phishing  
- Malware  
- Ataques de força bruta  
- Exploração de vulnerabilidades  

---

### 📖 Glossário

- **Firewall** → sistema de controle de tráfego de rede  
- **Phishing** → técnica de engenharia social para roubo de dados  
- **Malware** → software malicioso  
- **Exploit** → código que explora vulnerabilidades  
- **Pentest** → teste de invasão controlado
